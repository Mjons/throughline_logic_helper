# Template Authoring & Beat-by-Beat Agent Mode

## The Problems

### 1. The agent picks a framework and you're stuck with it

Right now the agent selects from 13 built-in frameworks. It picked Raskin 5-Beat for the Throughline pitch — a compressed cold-outreach format — when the user probably wanted the full 8-Beat (River, Gap, Fork, Promised Land, Troll, Magic Wand, Tide, Hoard) which has room for urgency, obstacles, and competitive framing. The 5-Beat exists for cold emails, not investor conversations.

**Why this happened:** The selector prompt describes the 5-Beat as "best for cold outreach, short pitches" and the 8-Beat as "best for full investor pitches, keynotes." But the context said "10-minute conversation with an angel" — which is ambiguous enough that the LLM could go either way. The user had no way to override without re-running.

**Fix:** Let the user pick or confirm the framework AND the specific beat count before generation starts. Show them all available options with beat counts visible.

### 2. Users can't create their own templates

Every template is either hardcoded in TypeScript or generated by the agent. There's no way to:

- Start with a blank canvas and define your own beats
- Clone an existing framework and customize it (add a beat, rename one, remove one)
- Build a hybrid (Raskin structure but with a custom "Demo" beat inserted)

### 3. No beat-by-beat agent mode

The current pipeline generates all beats in parallel — fast, but the user has no input per beat. A more collaborative mode would:

- Show the user one beat at a time
- Let them add context specific to that beat ("for the Troll beat, the biggest obstacle is regulatory, not technical")
- Generate options for that beat, let them pick/edit, then move to the next
- Build the throughline iteratively instead of all-at-once

## Plan

### Feature 1: Framework Picker UI

Replace the current "agent picks silently → confirmation card" flow with an explicit framework picker.

#### UI

A grid or list view in the IngestPanel, shown after the user has enough context (facts >= 3 or coverage >= 30%). Each framework is a card showing:

```
┌──────────────────────────────┐
│  Andy Raskin 8-Beat          │
│  8 beats · Full narrative    │
│                              │
│  River → Gap → Fork →        │
│  Promised Land → Troll →     │
│  Magic Wand → Tide → Hoard  │
│                              │
│  Best for: investor pitches, │
│  keynotes, full decks        │
│                              │
│  [AI recommended]            │
│  [Use this]                  │
└──────────────────────────────┘
```

The agent's recommendation is highlighted with a badge, but the user can pick any framework. An "AI recommended" tag appears on the one the agent would have chosen.

#### Data

```typescript
// New component
src / components / FrameworkPicker.tsx;

// Shows all frameworks from template-selector.ts
// Agent recommendation highlighted
// User clicks "Use this" → proceeds to generation
```

#### Flow change

```
Current:  Ingest → [Generate/LFG] → agent picks framework → confirm → generate all
Proposed: Ingest → [Generate] → framework picker (agent rec highlighted) → pick → generate
          Ingest → [LFG] → agent picks → generate (unchanged, still autonomous)
```

### Feature 2: Manual Template Builder

Let users create templates from scratch or by cloning an existing framework.

#### Entry points

1. **"New template" button** in the template dropdown or overview grid
2. **"Clone & edit" button** on any existing framework card in the picker
3. **"Customize" button** on a generated template to modify its beat structure

#### UI: Template Editor

A simple form-based editor (not the canvas — the canvas is for options, the editor is for structure):

```
Template name: [___________________________]
Description:   [___________________________]

Beats:
  ┌─ 1 ─────────────────────────────────┐
  │ Name:     [The River              ]  │
  │ Subtitle: [Name the undeniable shift]│
  │ Prompt:   [What changed?          ]  │
  │                        [↑] [↓] [✕]  │
  └──────────────────────────────────────┘
  ┌─ 2 ─────────────────────────────────┐
  │ Name:     [The Gap                ]  │
  │ ...                                  │
  └──────────────────────────────────────┘

  [+ Add beat]

  [Save template]  [Cancel]
```

Features:

- Drag-to-reorder beats (or up/down arrows)
- Add/remove beats (min 2, max 12)
- Each beat has name, subtitle, prompt — all editable
- Save creates a new template stored in localStorage
- Saved templates appear in the dropdown alongside static and generated ones

#### Data model

```typescript
// User-created templates stored separately
const USER_TEMPLATES_KEY = "throughline:user-templates";

// Same Template type — no changes needed
// Just stored in localStorage and merged into allTemplates
```

### Feature 3: Beat-by-Beat Agent Mode

A collaborative generation mode where the agent and user work through one beat at a time.

#### Flow

```
1. User picks framework (or creates custom)
2. Agent shows Beat 1 with its prompt
3. User can:
   a. Add beat-specific context ("for this beat, focus on AI regulation")
   b. Click "Generate" to get options for just this beat
   c. Skip to generate without extra context
4. Agent generates 3-5 options for Beat 1
5. User reviews, picks/edits, or regenerates
6. Click "Next beat" → repeat for Beat 2
7. Continue until all beats are populated
8. Template lands on canvas with full throughline
```

#### UI

The IngestPanel switches to a "stepper" mode:

```
┌─────────────────────────────────┐
│ Beat 2 of 8: The Gap            │
│ "What the shift didn't finish"  │
│                                 │
│ ┌─────────────────────────────┐ │
│ │ Add context for this beat:  │ │
│ │ [The main gap is that AI    │ │
│ │  tools generate images but  │ │
│ │  don't compose narratives]  │ │
│ └─────────────────────────────┘ │
│                                 │
│ [Generate options]  [Skip]      │
│                                 │
│ ── Generated options ────────── │
│ ○ Option A: ...                 │
│ ● Option B: ... (recommended)   │
│ ○ Option C: ...                 │
│                                 │
│ [← Prev]  [Next beat →]        │
│                                 │
│ ━━━●━━━━━━━━━━━━━━━━━━ 2/8     │
└─────────────────────────────────┘
```

The stepper shows:

- Current beat name, subtitle, prompt
- Optional textarea for beat-specific context
- Generate button (sends corpus + beat-specific context to generate just this beat)
- Generated options with radio selection
- Prev/Next navigation
- Progress bar at bottom

#### How it differs from current generation

| Dimension           | Current (LFG)             | Beat-by-Beat                   |
| ------------------- | ------------------------- | ------------------------------ |
| Speed               | ~8 seconds (parallel)     | ~5 sec per beat, user-paced    |
| User input per beat | None                      | Optional context per beat      |
| Review              | After all beats generated | After each beat                |
| Control             | Low (regenerate after)    | High (steer before generation) |
| Best for            | Quick first draft         | High-stakes pitches            |
